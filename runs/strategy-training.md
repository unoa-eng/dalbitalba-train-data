# dalbitalba 학습 데이터 고도화 전략: 학습 파라미터 관점 (Worker-2)

## 1. 현재 레시피 분석 및 적정성 평가
현재 설정: `r=64, alpha=64, use_rslora=True, lr=2e-4, seq_len=1024, 1epoch, bf16`

### 스타일 적응(Style Adaptation) 관점
- **LoRA Rank (r=64):** 8B 모델에서 r=64는 상당히 높은 수치입니다. 이는 단순한 지식 주입을 넘어 특정 커뮤니티의 '말투(Style)'와 '문법 파괴'를 학습하기에 적합한 용량입니다. rslora=True를 통해 안정성을 확보한 점은 긍정적입니다.
- **Learning Rate (2e-4):** LoRA CPT에서 2e-4는 표준적이지만, 한국어 베이스 모델(Qwen3)의 기초 능력을 보존하면서 '문체만 살짝' 입히기에는 다소 공격적일 수 있습니다. 만약 1차 루프에서 성능이 악화되었다면, LR을 1e-4로 낮추고 Epoch을 늘리는 방향을 고려해야 합니다.
- **Epoch (1):** 현재 데이터셋(약 4.6만 행)에 대해 1 Epoch은 데이터 노출 빈도가 낮은 도메인 용어(TC, 밀빵 등)를 모델이 '내재화'하기에는 부족할 수 있습니다.

## 2. CPT-only vs CPT+SFT 순차 학습 전략

### CPT-only (현재 budget30 계획)
- **장점:** 커뮤니티의 비정형 텍스트 구조와 토큰 분포를 밑바닥부터 학습하여 '자연스러운 문체'를 형성합니다.
- **단점:** 특정 질문에 대한 답변 능력(Instruction Following)이 부족하며, 광고성 문구의 패턴까지 학습할 위험이 큽니다.

### CPT+SFT 순차 학습 (권장)
- **전략:** 
    1. **CPT 단계:** LR을 조금 낮추고(1e-4) r=64로 문체와 용어를 충분히 노출시킵니다.
    2. **Merge:** CPT LoRA를 베이스에 병합합니다.
    3. **SFT 단계:** 정제된 (Hybrid 분리 및 광고 제거) Q&A 페어로 미세조정합니다. 이때 LR은 매우 낮게(5e-6 ~ 2e-5) 가져가서 문체를 고정하고 답변 형식만 가르칩니다.
- **이유:** CPT에서 배운 용어를 SFT에서 '어떻게 써야 하는지'를 알려주는 과정이 없으면, 모델은 용어는 알지만 광고처럼 뱉게 됩니다.

## 3. 학습 단계에서의 데이터 불균형 보정 전략

현재 핵심 용어(TC, 밀빵, 케어)가 광고에 쏠려 있는 문제를 학습 파라미터로 푸는 방법:

- **Loss Weighting (Weighted Language Modeling):** 도메인 용어가 포함된 시퀀스에 대해 Loss 가중치를 부여합니다. (예: 'TC', '밀빵' 단어가 포함된 라인에 2.0x 가중치)
- **Strategic Re-sampling (데이터단 협업):** 광고가 아닌 '일반 게시글' 중 해당 용어가 포함된 희귀 샘플을 5~10배 오버샘플링하여 학습 루프에 넣습니다. 이는 파라미터 조정보다 더 확실한 효과를 냅니다.
- **Curriculum Learning:** 
    - Phase 1: 전체 데이터(광고 포함)로 넓고 얕게 학습 (High LR, Short epoch)
    - Phase 2: 정제된 클린 데이터로 집중 학습 (Low LR, Long epoch)

## 4. Budget30 사이클의 유효성

- **결론:** **유효하지만 전략 수정 필요.** 
- 30달러 예산 내에서 L40S를 사용할 경우, 1 Epoch만 돌리기에는 시간이 남습니다(약 4~6시간 예상). 
- 남는 시간(Budget)을 활용해 **2~3 Epoch**을 돌리거나, **CPT 1.5 Epoch + SFT 1 Epoch** 형태의 하이브리드 구성을 추천합니다. 
- 단일 사이클로 끝내기보다는, **Low LR + High Epoch** 조합이 현재처럼 특정 용어가 부족한 상황에서 '암기'를 유도하기에 더 유리합니다.

## 5. 최종 제안 (Worker-2)

1. **LR 하향 및 Epoch 상향:** `LR=1e-4`, `Epoch=2`로 변경하여 저빈도 용어 노출 시간을 확보하십시오.
2. **LoRA Target 확대:** `all-linear`뿐만 아니라 `embed_tokens`와 `lm_head`를 LoRA 타겟에 포함하는 것을 검토하십시오. 한국어 신조어(Tokenization 특이성) 대응에 유리합니다.
3. **Hybrid SFT 병행:** CPT만 하지 말고, `cpt_corpus` 80% + `sft_pairs` 20% 혼합 학습을 수행하여 문체와 형식을 동시에 잡는 'Single-stage CPT+SFT'를 budget30 내에서 시도하십시오.
