## 2026 AI Evaluation SOTA 트렌드

### 1. LLM-as-a-Judge

**상위 트렌드**: 별도 LLM(Claude Opus, GPT-4)이 생성문 품질 평가.
- DeepEval: GEval, Faithfulness, AnswerRelevancy
- Ragas: 0 to 1 scale, 5개 metric 표준화
- Prometheus-2: open-source 13B judge model
- UK AISI Inspect AI: 정부 표준 안전 평가

**본 프로젝트 적용 방안**:
```ts
// 신규: apps/web/scripts/llm-judge.mjs
// Claude Opus를 judge로 사용
// "다음 2개 글 중 AI가 쓴 것은?" — 블라인드 판정
// score = 1 - (judge accuracy) = 탐지 회피율
```

### 2. Automated Regression

**트렌드**: 매 PR마다 자동 평가 + 이전 대비 점수 하락 감지.
- GitHub Actions에 measure-indistinguishability 통합 → JSON 비교
- Slack/Sentry 알림 (점수 -5%p 이상 하락)

**본 프로젝트 상태**: bot-quality-gate.yml 존재하나 indist 점수는 통합 안 됨.

### 3. Multi-detector Ensemble

**트렌드**: 단일 탐지기 대신 앙상블(KatFishNet + XDAC + HF detector).
- 각 탐지기별 점수 → 가중 평균
- 어떤 탐지기가 가장 예민한지 식별

**본 프로젝트 상태**: `detector.ts` 에 HF (Hello-SimpleAI/chatgpt-detector-roberta) 단일. KatFishNet 재현 모델 부재.

### 4. Prompt Engineering 표준

**트렌드**: Anthropic XML tag convention (prompt-injection 방어).
- `<context>`, `<instructions>`, `<example>` 명시적 구분
- Claude 4.5 공식 권장 패턴

**본 프로젝트 상태**: ✓ `<topic>`, `<context>`, `<title>`, `<body>` 래핑 적용 완료 (PR #6 cd92222).

### 5. Data Privacy for RAG (2026)

**트렌드**: **zero data retention** 의무화 (GDPR 강화 + 한국 개인정보법).
- 원문 저장 금지, fingerprint만
- 임베딩 공간 역복원 방지 (noise injection)

**본 프로젝트 상태**: ✓ style_corpus에 raw text 없음, fingerprint(JSONB) + embedding(vector) 만. **업계 모범 수준**.

## 추천 보강 순서
