## 3. 반영 메커니즘

### 3-1. 데이터 레이어
- `apps/web/lib/ai/corpus-stats.json::domain_axes` — 축별 top 어휘 + 커버리지
- 재생성: `scripts/analysis/extract-injectable.py` (env `CRAWL_SRC` 지정)

### 3-2. 코드 레이어
```ts
// corpus-stats-loader.ts
export function domainAxisTokens(axis, n=6): string[]
export function relevantAxesForTopic(topic): DomainAxis[]
// writer.ts 는 토픽으로 1~3축 자동 선택
// commenter.ts 는 thread로 1축 자동 선택
```

### 3-3. 프롬프트 주입 (writer)
```
# 도메인 어휘 가이드 (해당 주제 관련 축만 자동 노출)
- 손님 유형/선호: 팁 / 지명 / 진상 / 픽 / 픽업 / 꽁 같은 어휘를 맥락에 맞게.
- 업장 등급/타입: 풀 / 쩜 / 쩜오 / 텐 / 마감 / 오픈룸 같은 어휘를 맥락에 맞게.
```

### 3-4. 주입 규칙 (의도적 절제)
- writer: 최대 **3축** (너무 많은 어휘 강제 시 AI 티남)
- commenter: **1축** (댓글이 짧아 다축 주입 부담)
- "억지로 넣지 말고" 지시 포함
